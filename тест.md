- V2-C-19:

Минимальный Makefile:

```makefile
CC = gcc
CFLAGS = -Wall -Wextra
TARGET = program
OBJS = main.o utils.o

all: $(TARGET)

$(TARGET): $(OBJS)
    $(CC) $(CFLAGS) -o $(TARGET) $(OBJS)

%.o: %.c
    $(CC) $(CFLAGS) -c $< -o $@

clean:
    rm -f $(OBJS) $(TARGET)
```

Пояснение:
1. `all` — основная цель сборки.
2. `$(TARGET)` — цель для создания исполняемого файла из объектных файлов.
3. `%.o` — шаблонное правило компиляции `.c` в `.o`.
4. `clean` — фоническая цель для удаления объектных файлов и исполняемого файла.
5. Связь: каждый `.c` → компиляция → `.o` → линковка → исполняемый файл.
6. Фоническая цель `clean` нужна для очистки артефактов перед пересборкой.

- V2-C-20:

Функция безопасного преобразования строки в число:

```c
#include <stdio.h>
#include <stdlib.h>
#include <errno.h>

int safe_strtol(const char *prompt, long *result) {
    char buffer[64];
    char *endptr;
    long val;

    printf("%s", prompt);
    if (fgets(buffer, sizeof(buffer), stdin) == NULL) {
        return 0;
    }

    errno = 0;
    val = strtol(buffer, &endptr, 10);

    if (errno == ERANGE) {
        return 0;
    }
    if (endptr == buffer || *endptr != '\n') {
        return 0;
    }

    *result = val;
    return 1;
}
```

Пояснение:
1. Чтение строки через `fgets` с ограничением буфера.
2. Вызов `strtol` с сохранением `endptr`.
3. Проверка `errno == ERANGE` — выход за пределы диапазона.
4. Проверка `endptr == buffer` — ничего не распаршено.
5. Проверка `*endptr != '\n'` — наличие лишних символов после числа.
6. Возврат `1` при успехе, `0` при ошибке — без аварийного завершения.

Общий `impl<T>` и специализированный `impl Pair<u32>`:

```rust
struct Pair<T> {
    first: T,
    second: T,
}

// Общая реализация для любых T
impl<T> Pair<T> {
    fn new(a: T, b: T) -> Self {
        Pair { first: a, second: b }
    }
}

// Специализация для u32
impl Pair<u32> {
    fn sum(&self) -> u32 {
        self.first + self.second
    }
}
```

Пояснение:
1. Общий `impl<T>` содержит методы, работающие с любым типом `T`.
2. Специализированный `impl Pair<u32>` определяет методы только для конкретного типа `u32`, позволяя использовать операции сложения.
3. Практическая польза: отсутствие дублирования кода, сохранение типобезопасности, разделение общей и специфичной логики.

- V2-R-10:

Функция чтения `u32` из стандартного ввода с возвратом `Result`:

```rust
use std::io::{self, Write};

fn read_u32() -> Result<u32, String> {
    let mut input = String::new();
    print!("Введите число: ");
    io::stdout().flush().unwrap();

    io::stdin().read_line(&mut input)
        .map_err(|e| format!("Ошибка чтения: {}", e))?;

    let trimmed = input.trim();
    if trimmed.is_empty() {
        return Err("Пустая строка".to_string());
    }

    trimmed.parse::<u32>()
        .map_err(|_| format!("'{}' не является числом", trimmed))
}
```

Пояснение:
1. Чтение строки через `read_line`.
2. Очистка строки через `trim()`.
3. Проверка на пустую строку.
4. Вызов `parse::<u32>()` с преобразованием ошибки в строковое сообщение.
5. Возврат `Result` — вызывающий код обрабатывает ошибку без `panic!`.
